How Hash Tables Work
A hash table stores key-value pairs. It uses a hash function to convert a key into an index — O(1) on average. When two keys hash to the same index (a collision), we handle it with chaining (linked list at each bucket) or linear probing (try the next slot).
1
Hash function: Maps a key to an index. We sum the character codes of the key and take modulo table_size.
2
Collision — Chaining: If two keys hash to the same bucket, both are stored in a linked list at that bucket. Lookup scans the chain.
3
Collision — Linear Probing: If a bucket is occupied, try the next one (index + 1) % size, and so on until an empty slot is found.
4
Load factor: n/m (items / buckets). Keep it below 0.7 for good average performance. Resize the table when it gets too full.
