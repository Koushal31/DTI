How a Binary Search Tree Works
A BST is a binary tree where every node's left subtree contains only values smaller than the node, and the right subtree contains only larger values. This property makes searching O(log n) on average — each comparison eliminates half the remaining nodes.
1
Insert: Start at the root. If the new value is smaller, go left; if larger, go right. Repeat until you find an empty spot, then place the node there.
2
Search: Same logic as insert — compare with the current node and go left or right. If the value matches, you found it.
3
Delete: Three cases: leaf (just remove), one child (replace with child), two children (replace with inorder successor — smallest value in the right subtree).
4
Inorder traversal visits Left → Node → Right and produces values in sorted ascending order — a key property of BSTs.
