How Queues Work
A queue is a linear structure following FIFO — First In, First Out. Think of a line at a checkout counter. The first person to arrive is the first to be served.
1
Enqueue: Add an item to the rear of the queue. O(1) — no shifting needed.
2
Dequeue: Remove and return the front item. O(1) — the oldest item is processed first.
3
Peek: View the front item without removing it. O(1) — check what's next without changing the queue.
4
Overflow / Underflow: Enqueueing to a full queue or dequeueing from empty are errors to guard against.
